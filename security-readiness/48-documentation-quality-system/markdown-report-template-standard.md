# Markdown Report Template Standard

## Purpose
Standardize markdown report structure for consistent review, comparison, and audit traceability.

## Required Section Order
1. Title
2. Report Metadata
3. Scope
4. Method
5. Findings
6. Evidence References
7. Risks and Gaps
8. Recommendations
9. Residual Risk
10. Approvals / Review Notes

## Required Metadata Fields
- Report ID
- Prepared By
- Prepared Date (UTC)
- Version
- Classification
- Status

## Writing Rules
- Use concise, verifiable statements.
- Separate facts from assumptions.
- Link each security-relevant finding to one or more evidence references.
- Explicitly tag finding confidence as `Verified`, `Partially Confirmed`, or `Unknown`.

## Template Binding
All new client-facing and internal readiness reports should start from:
- `security-readiness/templates/standard-report-template.md`
