# Service Menu (Client-Facing)

## Purpose
This service menu outlines AI Trust & Security Readiness services for internal knowledge assistants using RAG and autonomous agent capabilities. Services are structured to support evidence-based launch decisions and ongoing governance.

## Scope Boundary
- In scope: readiness assessment, control design, verification planning, evidence packaging, and launch-gate decision support.
- Out of scope: production managed security operations, legal counsel, and guaranteed regulatory certification.

## Core Service Lines

### 1) Readiness Discovery & Scoping
- Stakeholder intake workshops
- AI system scope definition
- Trust boundary identification
- Assumptions and dependency mapping

**Deliverables**
- Scoped system profile
- Initial risk register
- Readiness workplan with phase boundaries

### 2) Threat Modeling & Abuse Case Development
- Threat modeling for RAG pipelines and agent toolchains
- Abuse-case development (identity boundary bypass, data exfiltration, prompt abuse)
- Attack path enumeration

**Deliverables**
- Threat model packet
- Abuse case catalog
- Prioritized mitigation backlog

### 3) Controls & Retrofit Planning
- Retrieval authorization controls
- Policy decision logging requirements
- Fail-closed control design
- Layer Retrofit plan and Secure Starter Kit integration guidance

**Deliverables**
- Control matrix with owners
- Retrofit roadmap
- Secure Starter Kit adoption plan

### 4) Verification, Red Teaming, and Evidence Pack
- Control test design and execution guidance
- Abuse-case testing scripts and acceptance criteria
- Evidence traceability mapping (code/config/test/log)

**Deliverables**
- Verification summary
- Evidence pack draft
- Residual risk statement

### 5) Launch Gate Decision Support
- Readiness score framework
- Launch-gate criteria workshop
- Decision memo facilitation

**Deliverables**
- Launch Gate Decision memo (Go / Conditional Go / No-Go)
- Open risks and remediation commitments
- Executive-ready final report

### 6) Governance & Continuous Review
- Operational runbook baseline
- Control ownership and cadence planning
- Continuous review checklist

**Deliverables**
- Governance operating model
- Quarterly review template
- Readiness drift watchlist

## Optional Add-On Services
- Third-party and supply chain risk deep dive
- Audit event and runtime tracing architecture
- Readiness dashboard design
- Retainer-based advisory support

## Commercial Packaging (Summary)
- Sprint package (targeted phase completion)
- Program package (end-to-end readiness lifecycle)
- Retainer package (ongoing governance and control verification)

## Evidence-First Commitment
All security and readiness claims are classified as:
- **Verified** (observed through evidence)
- **Partially Confirmed** (limited validation)
- **Unknown** (insufficient evidence)

No production readiness claim is made without documented evidence and explicit launch-gate criteria.
