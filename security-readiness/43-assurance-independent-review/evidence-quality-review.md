# Evidence Quality Review

## Objective
Assess whether readiness evidence is complete, reliable, current, and reproducible.

## Quality Gates
Evidence quality is insufficient if any gate below fails.

| Gate ID | Quality Gate | Pass Criteria | Result (`Pass` / `Fail`) | Notes |
|---|---|---|---|---|
| EQR-01 | Traceability | Security claim maps to code/config/test/assumption. |  |  |
| EQR-02 | Reproducibility | Commands are exact and can be rerun in defined environment. |  |  |
| EQR-03 | Freshness | Evidence date and commit align with release candidate. |  |  |
| EQR-04 | Integrity | Artifact source and modification chain are documented. |  |  |
| EQR-05 | Clarity | Findings clearly distinguish `Verified`, `Partially Confirmed`, `Unknown`. |  |  |
| EQR-06 | Redaction Hygiene | No secrets; sensitive data redacted as `[REDACTED]`. |  |  |
| EQR-07 | Contradiction Handling | Conflicting evidence is resolved or formally escalated. |  |  |
