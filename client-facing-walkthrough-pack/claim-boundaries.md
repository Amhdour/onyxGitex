# Claim Boundaries

## Allowed public wording
- “This is an AI Trust & Security Readiness portfolio and methodology project.”
- “It demonstrates evidence-based launch-gate thinking.”
- “It includes local evidence harnesses for RAG, observability, and agent runtime scenarios.”
- “It includes a client-specific production template.”
- “It does not claim production readiness without real client evidence.”

## Forbidden public wording and replacements
- “Production ready.” → “Template-ready pending client evidence.”
- “Fully secure.” → “Security posture assessed with documented residual risk.”
- “Client approved.” → “Client approval pending named signoff.”
- “Compliance approved.” → “Compliance review pending.”
- “Real production GO.” → “No production GO without client evidence and approval.”
- “Guarantees RAG safety.” → “Improves and verifies RAG controls with evidence.”
- “Guarantees agent safety.” → “Improves and verifies agent controls with evidence.”
- “Works safely for every client.” → “Requires client-specific validation and governance.”
- “Real LangGraph production proof.” → “Local agent runtime evidence model only.”
- “Real external integration proof.” → “External integrations require client-specific verification.”
