# Client Demo Pack

## Objective
Provide a structured demonstration of AI Trust & Security Readiness for an Onyx-based internal knowledge assistant.

## Demo Persona
- Fictional organization: **Northstar Biologics (DemoCo)**
- Use case: internal knowledge search and Q&A across controlled document sets

## Pack Contents
1. **Executive Brief (10–15 min)**
   - Business risk context
   - Target launch decision criteria

2. **System Walkthrough (20 min)**
   - Onyx architecture in demo scope
   - Retrieval and identity boundaries
   - Policy and tool control points

3. **Control Demonstration (25 min)**
   - Retrieval authorization checks
   - Fail-closed behavior examples
   - Output safety and misuse constraints

4. **Evidence Walkthrough (20 min)**
   - Test logs and result summaries
   - Observability and audit artifacts
   - Residual risk and unknowns

5. **Launch Gate Review (15 min)**
   - Readiness scoring interpretation
   - “Go / no-go / limited-go” options

## Deliverables to Client
- Scope and assumptions register
- Threat model and abuse-case package
- Control catalog and ownership map
- Test evidence bundle
- Launch gate memo with residual risks
- 30/60/90-day improvement roadmap

## Client Outcomes
By the end of the demo pack, stakeholders should be able to:
- Understand what has been validated vs not validated
- Evaluate security and governance readiness for a defined scope
- Decide next implementation and verification steps
